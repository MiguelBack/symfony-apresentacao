O que é?

Symphony é um poderoso framework full stack multiplataforma para desenvolvimento php de alta performance, ele é open source, usa a arquitetura MVC e conta com uma documentação extensa e bem detalhada.

Para que serve?

Serve para a aplicação dos princípios ágeis do desenvolvimento (DRY, KISS ou XP), com foco nas regras de negócio.

Vantagens e Desvantagens

    Vantagens

  • Flexibilidade;
  • Arquitetura MVC;
  • Documentação.

    • Desvantagens

  • Relativo maior nível de complexidade.

Instalação

  • Site oficial
  • Criar uma variável de ambiente com o caminho do diretório instalado
  • Verificar se foi instalado corretamente > symfony -V
  • Verificar se é possível rodar o framework >symfony check:requirements
  • Dentro do diretório para criar o projeto >symfony new SymfonyFirstApp --webapp
  • Iniciar o Apache no xampp
  • Iniciar o servidor Symfony (dentro do diretório do projeto) >symfony server:start
  • Acessar http://localhost:8000 (página padrão Symfony)

Códigos

Estrutura de Diretórios

  • public - front controller: o que é acessível pelo navegador (HTML, CSS, JS, imagens...);
  • config - arquivos de configurações
  • src - códigos PHP
  • vendor - dependências do projeto;
  • bin - execução de tarefas do projeto por meio do cmd (controladores, arquivos...);
  • migrations - estrutura do banco de dados;
  • templates - frameworks do projeto;
  • translations - traduções, caso o projeto tenha mais de um idioma;
  • var - permissões de escrita (cache.log);

Códigos

Criando um usuário...

  • >symfony console make:user
  • Informar nome da classe do usuário;
  • Informar se será salvo no banco de dados padrão;
  • Informar qual atributo será único para cada usuário (username, email, id...);
  • Informar se a senha será criptografada.
  • Pronto! Entidade usuário criada =)
